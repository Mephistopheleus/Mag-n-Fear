# АРХИТЕКТУРА V2: ГИБРИДНЫЙ МАТЕМАТИЧЕСКИЙ СКАЛЬПЕР (HYBRID MATH SCALPER)

## ГЛАВНАЯ ИДЕЯ
Переход от примитивного "голосования" анализаторов к глубокому синтезу рыночной модели. Система должна действовать как опытный трейдер, видящий структуру, контекст и взаимосвязи, а не просто набор сигналов.

## ПРИНЦИПЫ
1. **Глубокий Синтез**: Все данные (цена, объем, новости, стакан) объединяются в единую модель рынка (`MarketModel`).
2. **Контекстные Сценарии**: Торговые планы генерируются только в соответствии с общей картиной рынка.
3. **Адаптивность**: Система обучается на результатах (реальных и теневых), стремясь к идеалу (100% прибыль, 0% просадка).
4. **Асинхронность**: Параллельная обработка данных и исполнение сделок без блокировок.
5. **Разделение Ответственности**: Каждый модуль выполняет строго свою функцию.

## КОМПОНЕНТЫ СИСТЕМЫ

### 1. MarketSynthesizer (Мозг / Архитектор)
- **Задача**: Создание единой модели рынка (`MarketModel`).
- **Вход**: Данные от всех анализаторов, цены, объемы, новости.
- **Логика**: 
  - Определяет доминирующий тренд (Short/Mid/Long).
  - Вычисляет ключевые уровни поддержки/сопротивления (SR).
  - Оценивает "настроение" рынка (Aggression, Fear, Greed).
  - Синтезирует всё в объект `MarketModel`.
- **Выход**: `MarketModel` (объект с полями trend, levels, sentiment, volatility).

### 2. ProbabilityField (Матрица / Склад)
- **Задача**: Хранение "облака точек" прогнозов.
- **Структура**: Список объектов `PredictionPoint` {price, time, probability, source}.
- **Логика**: 
  - Принимает точки от анализаторов и RiskManager'а.
  - Не производит вычислений, не усредняет, не группирует.
  - Предоставляет доступ к данным для чтения.
- **Принцип**: "Тупой склад" — только запись и чтение.

### 3. ScenarioWriter (Стратег / Планировщик)
- **Задача**: Генерация торговых сценариев на основе `MarketModel`.
- **Вход**: `MarketModel` от Synthesizer, данные из ProbabilityField.
- **Логика**:
  - Создает сценарии для разных горизонтов: Scalp (быстро), Trap (разворот), Swing (тренд).
  - Рассчитывает параметры: Entry, StopLoss, TakeProfit, Leverage.
  - Фильтрует сценарии, противоречащие модели (например, покупка на сильном сопротивлении при медвежьем тренде).
- **Выход**: Список объектов `TradeScenario`.

### 4. RiskManager (Контролер / Страж)
- **Задача**: Фильтрация сценариев и управление рисками.
- **Вход**: `TradeScenario`.
- **Логика**:
  - Проверяет соответствие лимитам риска (MaxRiskPerTrade).
  - Оценивает соотношение R/R (Profit/Loss).
  - Принимает решение: **Real** (в работу), **Shadow** (в тень на обучение), **Reject** (отклонить).
  - **Умный Трейлинг**: Мониторит открытые позиции, двигает стоп-лосс, закрывает при нештатных ситуациях.
- **Выход**: Вердикт + скорректированные параметры стопа.

### 5. Executor (Дилер / Исполнитель)
- **Задача**: Физическое исполнение ордеров.
- **Режимы**:
  - **Dealer**: Отправляет ордера на биржу (Binance Testnet/Real).
  - **Shadow**: Симулирует исполнение локально, записывая результат для обучения.
- **Логика**: Строго исполняет приказы, не принимает решений.

### 6. AutoTuner (Инженер / Обучатель)
- **Задача**: Настройка параметров системы для достижения Идеала.
- **Идеал**: 100% Прибыль/День, 100% WinRate, 0% Просадка.
- **Логика**:
  - Анализирует "снимки" закрытых сделок (как реальных, так и теневых).
  - Сравнивает результат с идеалом.
  - Корректирует параметры в конфиге: `confidence_threshold`, `max_risk`, `leverage`, веса доверия к типам сигналов.
  - Работает в фоне, постоянно улучшая метрики.

## ПОТОК ДАННЫХ
1. **Сбор**: Анализаторы -> ProbabilityField.
2. **Синтез**: ProbabilityField + Данные -> MarketSynthesizer -> MarketModel.
3. **План**: MarketModel -> ScenarioWriter -> TradeScenarios.
4. **Фильтр**: TradeScenarios -> RiskManager -> Verdict (Real/Shadow).
5. **Исполнение**: 
   - Real -> Dealer -> Биржа.
   - Shadow -> ShadowDealer -> Локальный лог.
6. **Обучение**: Результаты сделок -> AutoTuner -> Конфиг.

## ЦЕЛИ РАЗВИТИЯ
- Реализовать полноценный `MarketSynthesizer`.
- Настроить глубокую связь между Сценаристом и Моделью рынка.
- Обеспечить непрерывное обучение через связку Теневик+Автотюнер.
- Достичь стабильной работы в асинхронном режиме без пауз.
